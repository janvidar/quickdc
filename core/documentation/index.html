Introduction

The QuickDC core library is the core component of QuickDC, which should be fairly portable and platform independent. The core is for this reason separated from the GUI, so it is possible to build on different platforms with different windowing systems, or build a client without a GUI at all if you like.

The whole API is designed to operate in an asynchronous environment, which means that you don't have to use multiple threads unless you want to. This also means that you will have to feed timeslices to the message loop once in a while so that it can dispatch events, otherwise it will not work.
